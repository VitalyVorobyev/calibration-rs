# B3C-APP-WIRING - Puzzleboard + ring-grid in the Run workspace

## Scope

Surface the two new detectors (B3C-PUZZLEBOARD, B3C-RINGGRID) in the desktop
app's Run workspace and verify the frontend builds against the regenerated
dataset schema.

## What the app needed (and didn't)

The Run workspace is **schema-driven for targets** and **gated only by
topology**:

- Target editing is driven by `app/src/schemas/dataset_spec.json` (the
  `TargetSpec` `oneOf`), regenerated by `cargo xtask emit-schemas` in the
  B3C-RINGGRID commit. The manifest form already renders the puzzleboard and
  ring-grid variants — including the realigned hex-lattice ring-grid fields —
  with no hand-written target-kind allowlist to extend (`topologies.ts` gates
  *topologies*, not target kinds; `targetKindOf` reads the kind generically).
- Detection dispatch is server-side: the Tauri runner calls the pipeline
  `build_*_input` → `pick_detector`, which now resolves `"puzzleboard"` and
  `"ringgrid"`. No Tauri or TS dispatch change was required.

So a user can author and run a puzzleboard or ring-grid manifest today via the
manifest editor; the run reaches the new detectors with no `UnsupportedTarget`.

## Files Changed

- `app/src/workspaces/RunWorkspace/presets.ts` — corrected the stale DS8
  preset `disabledReason` (it claimed "the puzzleboard detector is not wired
  into Run yet", which is no longer true). The card stays disabled because no
  DS8 puzzleboard manifest is committed (the DS8 data ships without a
  `dataset.toml`), and the reason now points users to the manifest editor.

No enabled puzzleboard/ring-grid *preset* was added: no such dataset manifest
is committed to the repo (the `puzzle_130x130` rig lives under the local-only
`/data/puzzle`, and there is no ring-grid dataset). Presets point at real
manifests, so adding a card with no backing data would be a broken entry.

## Validation Run

- PASS: `bun run build` (tsc + vite) — frontend compiles against the
  regenerated schema; only the pre-existing chunk-size advisory remains.
- `app/dist/` build artifacts are gitignored (placeholder `index.html` +
  `.gitkeep` restored); not committed.
- Playwright smoke is unchanged: it views exports (Diagnose) and does not
  exercise detector dispatch, so it is not a meaningful gate for this change;
  the Rust detector unit tests + dispatch tests cover the new path.

## Follow-Ups

- When a puzzleboard or ring-grid dataset is committed (or available locally),
  add an enabled preset and run the full detect → calibrate → Diagnose flow.
